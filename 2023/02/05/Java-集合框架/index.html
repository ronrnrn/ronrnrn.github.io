集合框架

Java中提供了很多集合类、接口和算法类，作为一个整体称为集合框架。

集合框架中有两个顶层接口，所有的集合类都是这两个接口的实现类型；

  • Collection：存储元素
  • Map：key-value

集合中存储的是对象的引用

Collection接口

概述

  1. Collection集合是一个管理一组对象的单一对象，在集合内的对象称之为元素(element)。
  2. Collection接口是List，Set，Queue接口的父接口
  • List：元素有序、可重复
  • Set：元素无序、不可重复
  • Queue：元素先进先出

List接口

概述

List代表一个元素有序、且可重复的集合，每个元素都有其顺序索引。

链表

LinkedList类是List接口的实现类
特点：

  • 非线程安全
  • 允许元素为null，允许元素重复
  • 插入删效率高，查找效率低
  • 基于链表实现，不存在容量不足的问题，没有扩容方法

Vector和Stack

Vector

Array和Vector是List接口的两个典型实现

  • Vector是过期的集合类，线程安全
  • ArrayList线程不安全

Stack

作用：

  • 完成悔棋功能
  • Ps，Word的撤销功能

Set

Set是Collection接口的子接口，是一个元素不重复的Collection，此类有3个重要的实现类：

  • HashSet-hash
  • LinkedHashSet-有顺序
  • TreeSet-排序，内部是树

HashSet

按Hash算法来存储集合中的元素，具有很好的存取和查找性能

特点：

  • 不能保证元素的排列顺序
  • HashSet不是线程安全的
  • 集合元素可以是null
